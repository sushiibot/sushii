sushii can automatically catch messages that post known Discord scam images,
like the "MrBeast" style deposit scams where images claim you can get a sum of
money for free from an online casino promotion (often impersonating other
influencers too). These scams spread without any links or text, so sushii will
check only images for this feature.

## What it does

When a member posts a message with one or multiple known scam images, sushii:

1. Times the member out for 10 minutes.
2. Deletes the messages.
3. Posts an alert to your alerts channel with the image and quick
   **Warn / Kick / Softban / Ban** buttons.
4. Logs it in your mod log, just like any other moderation action.

<Callout type="warn" title="sushii only catches images it already knows about">
This feature only works on scam images that have already shown up elsewhere and
been confirmed as scams. A brand new scam image won't get caught the first time
it appears anywhere.
</Callout>

<Callout type="info" title="False positives can happen">
It's rare, but a normal image could occasionally look close enough to a known
scam to get flagged by mistake. If that happens, please click the button in the
alert message to report it as a false positive.
</Callout>

## Setup

Enable it from **`/settings` → Automod**:

- **Known Scam Images** turns detection on. Off by default.
- **Alerts Channel** is where alerts and the moderation buttons show up.
  Optional, but recommended.
- **Exempt Roles** are roles this feature never touches (bots, trusted
  partners, etc).

sushii needs the **Timeout Members** and **Manage Messages** permissions to
take action. If it can't time someone out, for example if their role
outranks sushii's, it'll still delete the message and send the alert.

## Where the scam images come from

sushii keeps a shared list of known scam images built from what's been seen
across all public servers it's in. This runs in the background all the time,
whether or not you've turned the setting above on. This is what keeps the list
useful for every server running sushii, including yours.

sushii does not scan every image it sees. It first looks for behavioral
signals, such as the same images spreading rapidly across servers, and only
analyzes image content once that behavior crosses a high-confidence
threshold.

sushii does not keep a copy of your server's images for this. What it stores
is a short fingerprint of each image, just enough to tell if two images
match. You can't recreate or view the original image from that fingerprint.
